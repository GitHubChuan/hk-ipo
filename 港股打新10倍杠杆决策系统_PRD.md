# 港股打新 10 倍杠杆决策能力增强 PRD

## 1. 背景

当前网站已经具备港股打新的基础管理框架，包括：

- 新股日历
- 标的评估
- 历史回测
- 额度分配
- 持仓申购
- 卖出分润
- 历史复盘

从整体产品形态看，网站已经搭建出一条较完整的打新流程：

> 新股发现 → 标的评估 → 额度分配 → 实际申购 → 卖出结算 → 历史复盘

其中比较有价值的能力包括：

1. **标的评估模块**  
   可根据一手中签率、入场费、预期涨幅、红鞋系数，计算单手期望利润和推荐档位。

2. **收益回测模块**  
   内置 20 支近期 IPO 历史样本，支持搜索、机制筛选、排序，并展示盈利率、平均暗盘、平均首日、组合总收益、ROI 等指标。

3. **完整流程闭环**  
   已经形成从评估到复盘的主流程框架。

但如果目标是服务于一位希望**充分使用 10 倍资金杠杆**进行港股打新的投资者，当前系统仍然偏向“现金申购 / 中签后一手收益”视角，而不是“融资申购后的真实资金效率与风险”视角。

---

## 2. 目标用户

### 2.1 用户画像

经验型港股打新投资者 / 小型合伙制打新主理人。

### 2.2 用户特征

- 有固定本金池，例如 HK$50 万至 HK$300 万。
- 希望使用 5x–10x 孖展融资提高申购规模。
- 关注的不只是“这只新股会不会涨”，还包括：
  - 借钱打是否划算？
  - 10 倍杠杆后净收益是多少？
  - 融资成本会吃掉多少利润？
  - 如果破发，最大亏损是多少？
  - 多只新股并发时，资金优先给谁？
  - 哪个券商融资额度更适合使用？
  - 暗盘卖、开盘卖、首日卖哪个策略更优？
  - 合伙人之间如何分润、如何复盘？

## 3. 当前网站可用性判断

### 3.1 当前有用的部分

#### 3.1.1 标的评估有决策雏形

当前评估公式方向正确：

```text
赚钱期望 = 一手中签率 × 一手金额 × 预期涨幅
```

该公式可以初步回答：

- 这只票值不值得打？
- 单手期望收益大概是多少？
- 中签率和涨幅对结果影响有多大？

#### 3.1.2 历史回测有参考价值

当前内置 20 支近期 IPO，展示了：

- 盈利率
- 平均暗盘涨幅
- 平均首日涨幅
- 平均单手盈利
- 超购均值
- 组合总收益
- ROI

这些数据能帮助用户判断近期港股打新市场的整体热度。

#### 3.1.3 额度分配方向正确

当前额度分配模块的逻辑是：

> 并发期间把有限预算优先分配给赚钱期望最高的标的，剩余资金再用于分散摸一手。

该思路符合真实港股打新中的资金调度逻辑。

---

### 3.2 当前不足

#### 3.2.1 缺少“10 倍杠杆”显式建模

当前系统没有看到全局或单标的层面的以下字段：

- 杠杆倍数
- 融资比例
- 自有保证金
- 融资金额
- 预计冻结天数
- 融资年化利率
- 认购手续费
- 融资手续费
- 中签后需补款金额
- 杠杆后净利润
- 杠杆后 ROI
- 年化资金效率

这导致系统目前无法直接回答：

> 我用 10 倍杠杆打这只票，扣除融资成本后到底值不值得？

#### 3.2.2 回测口径偏乐观

当前回测更像是在回答：

> 如果中了一手，暗盘 / 首日卖出能赚多少？

但真实高杠杆打新更关心：

> 为了中这一手，我实际申购了多少金额？占用了多少保证金？借了多少钱？冻了几天？扣除融资成本后，资金年化收益是多少？

因此，现有 ROI 可能会高估实际资金效率。

#### 3.2.3 历史样本周期较短

当前历史样本集中在近期 IPO，且近期市场表现较好，容易造成乐观偏差。

需要关注以下风险：

- 新股周期转冷
- 暗盘高开低走
- 新股破发
- 中签率大幅下降
- 融资额度不足
- 高息融资吞噬利润

#### 3.2.4 缺少风险压力测试

对于 10 倍杠杆打新，最重要的不只是收益上限，而是亏损下限。

当前缺少以下情景测算：

- 暗盘 -5%
- 暗盘 -10%
- 暗盘 -20%
- 首日低开无法卖出
- 中签多手但破发
- 融资成本高于预期
- 多只新股同时冻结资金
- 最大单票亏损
- 最大组合回撤
- 保证金占用峰值

#### 3.2.5 实盘数据为空，无法验证闭环

当前真实账户数据为空：

- 在评标的：0
- 待分配候选：0
- 申购单：0
- 可结算标的：0
- 历史结算：0

因此目前只能验证产品框架，无法验证实盘运转效果。

#### 3.2.6 页面章节编号存在混乱

顶部导航与页面内部标题编号不一致，例如：

- Allocation 顶部是 §V，但页面内显示类似 §IV
- Holdings 顶部是 §VI，但页面内显示类似 §IV
- Settlement 顶部是 §VII，但页面内显示类似 §VI
- Review 顶部是 §VIII，但页面内显示类似 §VII

这会影响用户对流程的理解。

#### 3.2.7 资金口径不够清楚

首页显示：

```text
总资金池：HK$1,500,000
```

复盘模块显示：

```text
主理人本金：HK$500,000
占比：40%
```

两者之间缺少解释，例如：

- 总资金池是否包含杠杆？
- HK$1,500,000 是自有资金还是可动用总额度？
- 主理人 HK$500,000 占比 40%，其他 60% 来自哪里？
- 是否有未展示的合伙人？

## 4. 产品目标

### 4.1 核心目标

将网站从：

> 港股打新记录 / 评估工具

升级为：

> 面向 10 倍杠杆港股打新的资金效率与风险决策系统

### 4.2 关键问题

系统需要回答以下问题：

1. 这只新股是否值得用杠杆打？
2. 用几倍杠杆最划算？
3. 10 倍杠杆后，扣除融资成本的净期望收益是多少？
4. 如果破发，最多亏多少？
5. 多只新股同时招股时，资金优先给谁？
6. 哪个券商额度应该优先使用？
7. 暗盘、开盘、首日分别卖出的历史胜率如何？
8. 过去真实打新中，融资成本占收益比例是多少？
9. 当前组合的最大资金冻结峰值是多少？
10. 合伙人分润是否已经扣除了真实融资成本？

## 5. 需求范围

### 5.1 P0 需求

#### 需求 1：新增全局杠杆配置

##### 需求描述

在系统设置或资金配置区域增加“融资杠杆默认参数”。

##### 字段设计

| 字段 | 示例 |
|---|---:|
| 默认杠杆倍数 | 10x |
| 默认融资比例 | 90% |
| 默认年化融资利率 | 6.8% |
| 默认冻结天数 | 5 天 |
| 默认认购手续费 | HK$100 |
| 默认平台费 / 交易费 | HK$50 |
| 默认暗盘卖出手续费 | 可配置 |
| 默认现金安全垫 | 10% |

##### 验收标准

- 用户可以设置默认杠杆倍数为 1x、2x、5x、10x。
- 所有新股评估、额度分配、持仓申购默认读取该配置。
- 单只新股可以覆盖全局默认配置。
- 当用户选择 10x 杠杆时，系统自动计算融资金额和保证金占用。

#### 需求 2：标的评估增加“杠杆后收益测算”

##### 当前问题

当前只展示单手入场费、一手中签率、单手期望利润，未展示融资后的净收益。

##### 新增指标

| 指标 | 说明 |
|---|---|
| 现金申购金额 | 不加杠杆时所需资金 |
| 10x 可申购金额 | 自有资金 × 10 |
| 自有保证金 | 申购金额 / 杠杆倍数 |
| 融资金额 | 申购金额 - 自有保证金 |
| 预计融资成本 | 融资金额 × 年利率 × 冻结天数 / 365 |
| 预期中签手数 | 申购手数 × 中签率 |
| 中签后毛利润 | 中签手数 × 每手金额 × 预期涨幅 |
| 扣融资后净利润 | 毛利润 - 融资成本 - 手续费 |
| 杠杆后 ROI | 净利润 / 自有保证金 |
| 年化 ROI | ROI × 365 / 冻结天数 |
| 盈亏平衡涨幅 | 覆盖融资成本所需最小涨幅 |

##### 验收标准

- 输入预期涨幅、中签率、杠杆倍数后，系统实时计算净期望收益。
- 可以一键切换 1x / 5x / 10x。
- 推荐档位从“现金期望”升级为“融资后净期望”。
- 当融资成本高于预期收益时，系统明确提示“杠杆后不划算”。

#### 需求 3：历史回测增加“融资后真实 ROI”

##### 当前问题

历史回测展示的是中签后每手盈利和组合收益，不能代表杠杆申购真实收益。

##### 新增回测维度

| 维度 | 说明 |
|---|---|
| 假设杠杆倍数 | 1x / 5x / 10x |
| 假设融资利率 | 用户输入 |
| 假设冻结天数 | 用户输入 |
| 假设每票申购金额 | 用户输入 |
| 融资成本 | 自动计算 |
| 扣成本后收益 | 每票毛利 - 融资成本 |
| 净 ROI | 净收益 / 自有保证金 |
| 年化 ROI | 按冻结天数折算 |
| 最大亏损票 | 找出亏损最大样本 |
| 破发票影响 | 单独展示亏损样本 |

##### 验收标准

- 用户可以用历史 IPO 模拟“每只都用 10 倍杠杆打”的结果。
- 回测表格同时展示“毛收益”和“扣融资后净收益”。
- 如果某只票中签盈利很低但融资成本高，系统能显示净亏损。
- 汇总区增加：
  - 杠杆后总净利润
  - 融资成本合计
  - 融资成本占毛利比例
  - 杠杆后 ROI
  - 最大单票亏损
  - 最大连续亏损

#### 需求 4：额度分配支持多券商、多账户融资额度

##### 当前问题

当前只有本期可用预算和参与账户数，无法表达真实孖展打新的复杂约束。

##### 新增账户 / 券商模型

| 字段 | 示例 |
|---|---|
| 账户名称 | 富途-主账户 |
| 自有现金 | HK$500,000 |
| 可融资倍数 | 10x |
| 最大融资额度 | HK$5,000,000 |
| 当前已占用额度 | HK$1,200,000 |
| 可用额度 | 自动计算 |
| 融资利率 | 6.8% |
| 是否支持暗盘 | 是 / 否 |
| 是否红鞋优先 | 是 / 否 |
| 单票最大申购 | HK$2,000,000 |

##### 分配策略

支持三种分配模式：

1. **收益最大化**：优先分配给杠杆后净期望最高的新股。
2. **风险均衡**：限制单票资金占比，避免单只标的爆雷。
3. **摸一手优先**：对红鞋票或高中签率票优先分散摸一手。

##### 验收标准

- 系统可以根据多个券商账户的融资额度自动分配申购金额。
- 当某券商额度不足时，自动切换到其他账户。
- 分配建议必须展示原因，例如：
  - “该标的融资后年化 ROI 最高”
  - “该账户融资利率最低”
  - “该票破发风险较高，限制单票占比”

### 5.2 P1 需求

#### 需求 5：新增风险压力测试

##### 需求描述

每只新股评估时，自动展示不同涨跌幅情景下的杠杆后盈亏。

##### 情景设计

| 情景 | 假设 |
|---|---|
| 乐观 | 暗盘 +30% |
| 基准 | 用户预期涨幅 |
| 保守 | 暗盘 +5% |
| 破发 | 暗盘 -5% |
| 极端 | 暗盘 -20% |

##### 每个情景展示

- 毛利润
- 融资成本
- 净利润
- 自有资金亏损比例
- 是否触发风险警告

##### 验收标准

- 用户可以看到 10 倍杠杆下不同涨跌幅对应的盈亏。
- 当极端情景亏损超过自有资金一定比例时，红色提示。
- 可以设置单票最大可承受亏损，例如 HK$20,000。

#### 需求 6：卖出策略加入历史胜率参考

##### 当前问题

卖出策略模块目前没有足够数据验证策略质量。

##### 新增能力

对每只已上市新股展示：

- 暗盘卖收益
- 开盘卖收益
- 首日收盘卖收益
- 次日卖收益，如有数据
- 历史同类机制平均走势
- 当前暗盘涨幅是否已达到止盈线
- 是否建议立即卖出

##### 策略示例

| 条件 | 建议 |
|---|---|
| 暗盘涨幅 > 30%，且融资成本已覆盖 5 倍以上 | 暗盘卖出 70% |
| 暗盘涨幅 5%–15% | 开盘观察 |
| 暗盘破发 | 首日止损，不恋战 |
| 机制 B + 超购极高 + 暗盘爆涨 | 分批止盈 |

##### 验收标准

- 持仓标的上市前后，系统能根据暗盘 / 开盘 / 首日数据给出卖出建议。
- 建议必须显示理由。
- 结算时能记录实际卖出价格，并与建议策略做复盘对比。

#### 需求 7：复盘增加杠杆策略指标

##### 新增指标

| 指标 | 说明 |
|---|---|
| 累计毛利润 | 未扣成本收益 |
| 累计融资成本 | 所有融资成本合计 |
| 累计净利润 | 扣成本后收益 |
| 融资成本占毛利比例 | 判断杠杆成本是否过高 |
| 平均资金冻结天数 | 衡量资金周转效率 |
| 杠杆后年化 ROI | 真实资金效率 |
| 最大单票亏损 | 风险控制 |
| 最大回撤 | 组合风险 |
| 暗盘卖胜率 | 卖出策略评估 |
| 首日卖胜率 | 卖出策略评估 |
| 策略偏差 | 实际卖出 vs 系统建议 |

##### 验收标准

- 每笔结算必须扣除融资成本后进入复盘。
- 可以按月份、机制、券商、标的状态筛选。
- 可以看到“赚钱来自选股、杠杆，还是卖出纪律”。

##### 验收标准

- 抓取失败时有明确错误提示。
- 数据过期时有醒目提示。

## 6. 推荐实施优先级

### 第一阶段：让 10 倍杠杆可算清楚

优先做：

1. 全局杠杆配置
2. 标的评估增加融资后净收益
3. 历史回测增加融资成本后 ROI
4. 首页资金口径解释

阶段目标：

> 让用户能回答：这只票用 10 倍杠杆打，扣完成本到底赚不赚？

### 第二阶段：让资金分配更接近实盘

优先做：

1. 多账户 / 多券商融资额度
2. 智能额度分配
3. 风险压力测试
4. 资金冻结日历

阶段目标：

> 让用户能回答：多只票同时来，我的钱和券商额度应该怎么排？

### 第三阶段：让卖出和复盘形成策略闭环

优先做：

1. 暗盘 / 首日卖出策略建议
2. 实际卖出记录
3. 杠杆后复盘指标
4. 策略偏差分析

阶段目标：

## 7. 核心计算公式

### 7.1 融资成本

```text
融资成本 = 融资金额 × 年化融资利率 × 冻结天数 / 365
```

### 7.2 申购金额结构

```text
自有保证金 = 申购金额 / 杠杆倍数
融资金额 = 申购金额 - 自有保证金
```

### 7.3 预期中签手数

```text
预期中签手数 = 申购手数 × 一手中签率
```

### 7.4 毛期望利润

```text
毛期望利润 = 预期中签手数 × 每手金额 × 预期涨幅
```

### 7.5 净期望利润

```text
净期望利润 = 毛期望利润 - 融资成本 - 认购手续费 - 交易手续费
```

### 7.6 杠杆后 ROI

```text
杠杆后 ROI = 净期望利润 / 自有保证金
```

### 7.7 年化 ROI

```text
年化 ROI = 杠杆后 ROI × 365 / 冻结天数
```

### 7.8 盈亏平衡涨幅

```text
盈亏平衡涨幅 = 总成本 / 预期中签市值
```

### 8.1 当前网站是否有用？

有用。

还不够。

- 10 倍杠杆后扣成本赚多少？
